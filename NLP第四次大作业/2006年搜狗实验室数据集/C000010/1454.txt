　　管理，从ITIL开始
　　今天，在诸多服务管理的最佳实践指南之中，ITIL已在全球风潮渐起，原因在于其对于核心IT运作流程，诸如能力管理、可用性管理、事故管理、问题管理、变更管理、发布管理、配置管理和财务管理来说，是一系列最佳实践和标准方法论的集合。ITIL结合流程、人员和技术三大要素，为企业的IT建设提供一套从计划、研发、实施到运行维护的最佳实践框架。现在的CIO们发愁的已经不是如何为公司搭建一个系统，购买几台服务器，开发几个应用。而是数量众多的服务器，各种各样的应用，无数个系统，既独立又关联，纷繁复杂交织在一起；公司的业务需求不断地变化，新的要求不断涌现，原有的系统应用很快就不能满足需求，甚至常常被迫彻底改头换面。
　　越来越多的CIO们已经意识到这个问题，越来越多的企业也开始探讨这个问题。什么能让企业处变不惊？什么能让企业随需而动？
　　ITTL是一个答案。 而NetIQ公司知识型的IT服务管理解决方案为这个答案做出了有力的补充-NetIQ作为一家有十几年历史的全面的服务管理解决方案专业提供商，多年来持续地研发并不断完善着一套符合ITIL、ISO17799以及其他诸多最佳实践指南的完整的系统和安全管理解决方案。其知识型的服务管理解决方案（NetIQ AppManager性能与可用性管理解决方案、Security Manager安全管理解决方案、Vulnerability Manager配置及弱点管理解决方案以及Security Administration Suite变更管理解决方案……）将一系列最佳实践以知识的形式浓缩在其产品解决方案中，支持并实现了ITIL中的若干流程，帮助企业提高业务的持续性及运营效率。
　　八大特性，实现IT服务管理
　　ITIL的流程包括10个方面，将服务的交付与支持的要求拓展到更广的范畴。
　　在服务交付的过程中，服务水平、应用以及系统能力管理至关重要。而在服务支持过程中，事故、问题、配置和变更管理都是重要的组成部分。NetIQ AppManager 通过监测、分析、报告、以及策略执行等功能可以直接支持ITIL中的这些关键过程，从而充分满足IT服务管理的需求，加快问题的处理速度。
　　能力管理 为保证IT基础设施的总承载能力以最合理的成本投入、最适当的时机和方法与不断发展的业务需求相匹配。NetIQ AppManager可以监控并报告出很多IT设施及业务服务的使用状况（诸如磁盘空间使用情况及CPU利用率等），而这些都是对系统承载容量进行管理的必要信息。
　　系统可用性管理 AppManager的数据收集功能可以用来衡量技术维护与支持的质量。从而优化IT设施的承载能力，以实现最优的成本投入及持续稳定的可用性级别。
　　服务级别管理 AppManager Response Time模块可以用来收集数据以判断服务水平是否符合服务级别协议（SLA）的标准。并提供一系列流程，从计划、调整、监控到按照服务级别协议标准做出报告。
　　在安全管理方面，传统意义的“安全”仅限于在操作的层面进行管理，这导致没有能满足企业需求的适当、有效的安全控制方法可供选择。而安全管理作为ITIL中非同寻常的重要流程，已成为保障业务运营的关键内容。安全管理所实现的一个核心目标是管理企业的信息通讯技术（ICT），也就是管理那些提供信息和服务的网络、系统及软件。而ITIL的流程依赖于ICT来提供恰当、准确、适时的信息，企业据此易于做出正确的决策。既可以保证对日常运营的服务支持，也可以帮助确定并实现长期计划，改进IT服务。值得一提的是：在安全管理方面，NetIQ安全管理解决方案可协助企业在日常的IT服务的运营和支持中，不断适应ITIL的要求，管理风险并保证在IT服务管理的计划以及运营过程中对安全充分予以考虑。
　　事故管理NetIQ Security Manager能够对IT方面的安全事故进行实时监控和通知，还能识别出系统异常并做出自动化响应，从而帮助企业在事故管理方面做到更为提前和主动。NetIQ Vulnability Manager也可以帮助企业对事故进行提前侦测，主要是对配置的异常变化进行识别，找到偏离既定标准或是违背既定策略之处。将服务尽快恢复正常运行状态，并将事故对业务运行造成的不利影响降至最低。
　　问题管理NetIQ Security Manager不仅包括日志分析功能，而且具备定向跟踪辨别的能力，可以帮助IT部门对故障进行研究和诊断。与产品中可扩展的安全知识脚本相结合，这些定向跟踪辨别的能力可以帮助IT人员加快解决问题及恢复的速度，使故障恢复的时间最小化。另外，在解决问题过程中，NetIQ Vulnerability Manager能将最佳实践中的推荐数据及方法加以应用以便恢复服务，并使其符合安全及相关规范和策略的要求，从而帮助IT人员判断IT故障发生的根源并迅速解决问题。
　　配置管理NetIQ的解决方案可识别、控制、维护并确认企业IT系统中的服务及资产的配置。NetIQ Vulnerability Manager通过自动化地发现IT设施中所连接的设备，能够在配置发生了不符合既定标准的变化时向管理员发出通知。与此相仿，NetIQ Security Manager提供防侵入技术，以加强管理员对设备配置进行控制，保证配置的变更只能由获得授权的专人来完成。对于基于Windows的配置管理，Security Administration Suite提供一个集中控制点，依据组策略、AD的目录及Windows账户来管理和审计。
　　变更管理是最让服务支持人员头疼的。NetIQ的Security Administration Suite在Windows环境中提供了高水平的管理，使IT部门可以对组策略和账户管理中的变化进行评估及报告，并在变化实施之前强制执行审批流程。为了对变更进行检查验证，NetIQ Vulnerability Manager可以通过一系列范围广泛的指标，基于安全标准对变更进行报告，以确认服务仍旧符合企业内外部相关的规范和政策，从而尽快地恢复正常的系统运行服务，以尽可能减少对业务运营的影响。
　　发布管理NetIQ Security Administration Suite提供一套工作流来确保在基于Windows的环境中只有经过认可的变更才可以在生产环境中实施。这些变更包括账户修改、AD目录更新以及改变组策略。NetIQ Patch Manager也提供了一套粒度级的集中控制区，用于在IT系统中分发更新的软件，并对补丁的部署情况进行全面的报告，以保证生产环境中实施的配置都是经过认可的。另外，ITIL中“服务台”是帮助促进服务支持流程的一项功能。NetIQ 安全管理解决方案可帮助IT部门加强工作流程，保障知识的安全的同时，向服务台的员工进行报告，以帮助IT部门更加及时、有效地做出决策，使管理更为积极主动。
　　总体来说，NetIQ 系统与安全服务管理解决方案，加速了服务和支持流程，提高了生产率，使IT同业务发展紧密协调。NetIQ 服务管理解决方案自发布以来，已在国内得到了广泛的应用，覆盖银行、保险、电信、交通、政府、制造业、快速消费品行业等多个领域。国内客户如中国民航信息网络公司、广深铁路、河北网通、北京电报局、河北中行、石家庄商行、中信实业银行卡中心、国民保险公司、中外运集装箱公司、上海市公安局、大金空调、霍尼韦尔公司、玫琳凯化妆品公司等。企业在IT服务管理上的投资很快得到了满意的回报。
